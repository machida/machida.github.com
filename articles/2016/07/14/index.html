Google Photo の画像が middleman で作ったブログのローカルで表示されなくなった

このブログは middleman で作ってて、画像は Google Photo に保存した画像を貼ってます。いつものようにローカルで middleman を立ちあげてブログを書いてたら、突然画像が表示されなくなってしまいました。それの回避方法の説明です（Mac のみ）。

Google Photo を表示させる方法を一言で言うと、http://localhost:4567 でローカルの middleman をブラウザで表示しているのを http://192.168.xx.xx:4567 とプライベート IP アドレスで接続すれば表示されるようになります。

これだけなのですが、せっかくなのでプライベート IP アドレスの調べかたも書いておきます。

プライベート IP アドレスを調べる方法

ターミナルに ifconfig と打つと、

1
$ ifconfig

ドバーっと文字が出てきます。その中から「192.168.xx.xx」という数字を探します。MacBookPro 、MacBookAir を使っていて wifi でネットにつないでる場合は en0: のブロックにその数字があります（機種によって異なります）。

http://192.168.xx.xx:4567 にブラウザからアクセスすると、Google Photo の画像が表示されてるはず。

ちなみに、先ほども書きましたが、MacBookPro 、MacBookAir を使っていて wifi でネットにつないでる場合は en0: のブロックにその数字があるので、

1
$ ipconfig getifaddr en0

と、ターミナルに打つと、

1
192.168.xx.xx

こんな風にプライベート IPアドレスが表示されます。

システム環境設定から IP アドレスを調べる方法

プライベート IP アドレスを調べるもう一つの方法として、ターミナルを使わないで Mac のシステム環境設定から調べる方法があります。

システム環境設定

システム環境設定を開いて、「ネットワーク」を選択。

ネットワーク

「ネットワーク」の画面の中の、今ネットをつないでいる接続方法のところを見ると（キャプチャでは wifi）、そこにプライベート IP アドレスが表示されています。

ちなみに

http://192.168.xx.xx:4567 の URL で同じ LAN 内のスマホや、VirtualBox や VMware や Parallels Desktop で入れてる Mac の中の Windows からもアクセスできますので、middleman を実機確認するときにも便利。
